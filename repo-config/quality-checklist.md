# Skill Quality Checklist

Use this checklist before marking a skill as stable.

## Structure

- [ ] Skill has a clear name and ID
- [ ] Skill has a purpose
- [ ] Skill defines when to use it
- [ ] Skill defines when not to use it
- [ ] Inputs are described
- [ ] Outputs are described
- [ ] Instructions are actionable
- [ ] Constraints are explicit
- [ ] Examples are included
- [ ] Test cases are included

## Reasoning Quality

- [ ] Skill separates facts from assumptions
- [ ] Skill handles uncertainty
- [ ] Skill avoids unsupported claims
- [ ] Skill explains tradeoffs where relevant
- [ ] Skill asks clarifying questions only when necessary

## Safety and Reliability

- [ ] Skill avoids overclaiming
- [ ] Skill avoids unsafe advice
- [ ] Skill does not request unnecessary sensitive data
- [ ] Skill includes domain-specific limitations
- [ ] Skill recommends human review where appropriate

## Integration Readiness

- [ ] Skill can be copied into a GPT instruction
- [ ] Skill can be tested with example prompts
- [ ] Skill has a stable output format
- [ ] Skill has at least three realistic examples
- [ ] Skill has at least three test cases
